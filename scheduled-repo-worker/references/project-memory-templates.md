# Project Memory Templates

Use these templates for `project-memory` or `hybrid` mode.

## docs/WORK_STATE.md

```md
# Work State

## Current Status
<one short paragraph>

## Current Strategic Focus
- <focus 1>
- <focus 2>

## Just Completed
- <recent completed work>

## In Progress
- <current in-progress slice>

## Blockers
- <blocker or "No blocker right now.">

## Next Recommended Task
- <best next concrete task>

## Testing Status
- <tests run or "No relevant tests yet.">

## PR Status
- <current PR state>
```

## docs/NEXT_ACTIONS.md

```md
# Next Actions

Keep this file short, concrete, and execution-ready.

## Highest Priority
1. <next concrete task>
2. <next concrete task>
3. <next concrete task>

## After That
4. <later task>
5. <later task>

## Rules for Updating This File
- Remove completed tasks.
- Promote the next best concrete task.
- Do not let this become a brainstorm list.
```

## docs/PROJECT_LOG.md

```md
# Project Log

## YYYY-MM-DD
- <what changed>
- <tests run>
- <whether PR created>
- <next step>
```

## Prompt expectations
- normal runs should update all three files after meaningful work
- PR runs should update them before final PR/no-PR decision
- do not let these files replace real product work
